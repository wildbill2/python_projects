<h2>NVD Data Feeds</h2>

<div class="bs-callout bs-callout-danger"><strong class="h4Size">NOTICE</strong>

<hr />
<p>It is assumed that users of the data feeds provided on this page have a moderate level of understanding of the XML and/or JSON standard and XML or JSON related technologies as defined by www.w3.org. Currently, the NVD provides no other specific tools or services for processing vulnerability data.</p>
</div>

<p class="lead">The entire NVD database can be downloaded from this web page for public use. All NIST publications are available in the public domain according to Title 17 of the United States Code, however acknowledgement of the NVD when using our information is always appreciated.</p>
<strong class="h4Size">The following feeds are available:</strong>

<ul>
	<li><a href="#JSON_FEED">JSON Vulnerability Feeds</a>&nbsp;&ndash; Each vulnerability in the file includes a description and associated reference links from the CVE&reg; dictionary feed, as well as CVSS base scores, vulnerable product configuration, and weakness categorization.</li>
	<li style="display:none;&gt;&lt;a  data-cke-saved-href=">STIX Vulnerability Feeds&nbsp;&ndash; NVD data using&nbsp;the Structured Threat Information Expression (STIX&trade;)&nbsp;format.</li>
	<li><a href="/vuln/data-feeds#cpeMatch">CPE Match Feed</a>&nbsp;&ndash; A feed that provides the product/platform applicability statement to CPE URI matching based on the CPEs in the official CPE dictionary.</li>
	<li><a href="/vuln/data-feeds#RSS">RSS Vulnerability Feeds</a>&nbsp;&ndash; An eight day window of security related software flaws.</li>
	<li><a href="/vuln/data-feeds#transxml">Vulnerability Translation Feeds</a>&nbsp;&ndash; Translations of vulnerability feeds.</li>
	<li><a href="/vuln/data-feeds#comments">Vulnerability Vendor Comments</a>&nbsp;&ndash; Comments provided by vendors regarding a particular flaw affecting within a product.</li>
	<li><a href="/products/cpe">CPE Dictionary</a>&nbsp;&ndash; dictionary containing a list of products.</li>
	<li><a href="/config/cce">Common Configuration Enumeration (CCE) Reference Data</a>&nbsp;&ndash; Reference data for common configuration items.</li>
	<li><a href="/vuln/data-feeds#NCP_FEED">National Checklist Program (NCP) Checklists</a> &nbsp;&ndash; A&nbsp;list of all of the checklists categorized by the NCP.</li>
</ul>
<strong class="h4Size">How to keep up-to-date with the NVD data</strong><br />
The main&nbsp;vulnerability feeds provide CVE&reg; data organized by the first four digits of a CVE&reg; identifier except for the 2002 feeds which include vulnerabilities prior to and including &quot;CVE-2002-&quot;.&nbsp; Each feed is updated only if the content of that feed has changed. For example the 2004 feeds will be updated only if there is an addition or modification to any vulnerability with a starting CVE&reg; identifier of &quot;CVE-2004-&quot;. In addition, the &quot;recent&quot; feeds are a list of recently published vulnerabilities and the &quot;modified&quot; feeds are a list of recently published and modified vulnerabilities where &quot;recently&quot; and &quot;modified&quot; are defined as the previous eight days. These feeds are updated approximately every two hours.<br />
&nbsp;
<p>If you are locally mirroring the NVD data, the data feeds should be used to stay synchronized. After performing a one-time import of the complete data set using the compressed XML/JSON vulnerability feeds, the &quot;modified&quot; feeds should be used to keep up-to-date. The META file should be used to determine if a given feed has been updated since your last import. This helps prevent unnecessary downloads of the .zip or .gz files and should result in a reasonable use of less than 200 requests per day.</p>
<strong class="h4Size">META Files</strong>

<p>In addition, each of the data feeds is described by an associated plain text file with the same name as the .xml file with a .meta extension. These files are updated approximately every two hours to reflect changes within their respective feed file. For example, if the name of the file is nvdcve-2.0-Modified.xml then the .meta file name will be nvdcve-2.0-Modified.meta. The .meta file contains information about the specific feed including the last modified date and time, the size of the&nbsp;file uncompressed, and a SHA256 value of the uncompressed file:</p>

<div class="bs-callout bs-callout-success">
<p>lastModifiedDate:2015-09-10T08:40:09-04:00<br />
size:1273382<br />
zipSize:91619<br />
gzSize:91477<br />
sha256:ac782e2db403e2b09ad5dd676501e8755fda3f2bef347b7503491700c6c5eaff</p>
</div>

<h3 id="RSS">RSS Vulnerability Feeds</h3>

<p>NVD provides two RSS 1.0 data feeds. The first feed, <a href="https://nvd.nist.gov/feeds/xml/cve/misc/nvd-rss.xml">nvd-rss.xml</a> (<a href="https://nvd.nist.gov/feeds/xml/cve/misc/nvd-rss.xml.zip" target="_blank">zip</a> or <a href="https://nvd.nist.gov/feeds/xml/cve/misc/nvd-rss.xml.gz" target="_blank">gz</a>), provides information on all vulnerabilities within the previous eight days. The second feed, <a href="https://nvd.nist.gov/feeds/xml/cve/misc/nvd-rss-analyzed.xml">nvd-rss-analyzed.xml</a> (<a href="https://nvd.nist.gov/feeds/xml/cve/misc/nvd-rss-analyzed.xml.zip" target="_blank">zip</a> or <a href="https://nvd.nist.gov/feeds/xml/cve/misc/nvd-rss-analyzed.xml.gz" target="_blank">gz</a>), provides only vulnerabilities which have been analyzed within the previous eight days. The advantage of the second feed is that we are able to provide vulnerable product names in the title. The advantage of the former is that you learn about new vulnerabilities as soon as possible.</p>
